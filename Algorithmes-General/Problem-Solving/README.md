## 📝 Structure d'une Solution

Chaque problème contient :

```
nom-du-probleme/
├── README.md           # Énoncé + lien
├── solution.py         # Solution commentée
├── tests.py            # Tests unitaires
└── notes.md            # Approches alternatives
```

### Template README :
```markdown
# Titre du Problème

**Difficulté :** Easy/Medium/Hard  
**Source :** [LeetCode](url)  
**Tags :** Array, Hash Table, Two Pointers

## Énoncé
[Copier l'énoncé]

## Approche
[Expliquer l'approche]

## Complexité
- Temps : O(n)
- Espace : O(1)

## Solution
Voir `solution.py`
```

## 🎯 Méthodologie

### 1. Comprendre le Problème
- Lire l'énoncé attentivement
- Identifier les entrées/sorties
- Repérer les contraintes

### 2. Exemples
- Tester avec les exemples donnés
- Imaginer des cas limites
- Cas vides, négatifs, extrêmes

### 3. Approche
- Brute force d'abord
- Optimiser ensuite
- Utiliser des structures de données appropriées

### 4. Coder
- Écrire proprement
- Commenter les parties complexes
- Nommer les variables clairement

### 5. Tester
- Valider avec tous les cas
- Vérifier les edge cases
- Analyser la complexité

### 6. Optimiser
- Peut-on faire mieux ?
- Compromis temps/espace
- Solutions alternatives
